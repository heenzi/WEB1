What's the...



1. Repatory : 저장소, 한 프로젝트 당 하나의 레파토리(폴더) 할당
2. README : 레파토리 설명 파일(별도로 설명서를 만든 느낌)
3. Branch : 한 레파토리 안에서 다양한 테스트를 하기 위한 가지들.
main branch말고 다른 브랜치를 만들어 파일을 수정하면 메인 파일은 그대로 남기고 일부만 수정할 수 있다.
파일 수정 중 오류가 생겨 날리는 걸 방지하기 위해/여러가지 방면으로 수정하는 것을 모두 남기기 위해 쓰임.
4. commit : 변경사항이 생겼을 때 무얼 변경, 수정했는지 적는 것.
